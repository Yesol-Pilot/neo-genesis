# 올바른 AiNo 좌파 지지 콘텐츠 마스터 기획

작성일: 2026-05-24 KST
채널: 올바른 AiNo / @leftaino
적용 범위: TikTok, YouTube Shorts, Instagram Reels용 정치 시사 숏폼 생성 파이프라인

이 문서는 기존 `WORKFLOW_DESIGN_SPEC.md`, `PIPELINE_DESIGN.md`, JSON 전략 설정을 대체하지 않는다. 목적은 좌파·진보 지지층 대상 콘텐츠의 기획 품질 기준을 명확히 하고, 이후 코드와 설정 변경의 기준점으로 쓰는 것이다.

## 1. 결론

올바른 AiNo는 노골적인 선거운동 채널이 아니라, 진보 지지층이 분노하고 공유하고 댓글로 맞붙게 되는 공공 이슈 해설 채널로 설계한다.

핵심 방향은 다음 4개다.

1. 진보층의 감정 트리거: 공정성, 역사 기억, 책임 회피, 특권, 혐오 방치, 검찰·권력 남용.
2. 보수 반론의 의도적 배치: 우파 주장을 1개 이상 넣어 댓글 논쟁을 만든다.
3. 출처 기반 자극성: 보도·여론조사·재판 일정·당사자 발언을 기준으로 강하게 말하되, 확인 안 된 범죄 단정은 피한다.
4. 수익화 최적화: 1분 이상 원본성, 검색 가치, 시청 지속, 댓글·공유를 중심으로 만든다.

금지 방향은 다음이다.

- 직접적인 투표 요청, 후보 지지·반대 광고 문구.
- 허위 여론조사, 선거 절차·결과 관련 미확인 주장.
- 실존 정치인 얼굴 합성, 정당 로고 합성, 실제 언론 화면처럼 보이는 조작 이미지.
- 같은 주제·같은 문구·같은 배경의 반복 게시.

## 2. 채널 포지셔닝

### 한 줄 정의

진보 지지층을 위한 영화 같은 정치 쟁점 해설 카드뉴스.

### 시청자 감정

- "이건 그냥 넘어가면 안 되지."
- "반론까지 들어도 결국 책임 문제 아닌가?"
- "댓글에서 한 번 붙어볼 만하다."
- "다음 편도 봐야겠다."

### 말투

- 따뜻한 교양형보다 한 단계 더 세게 간다.
- 단정한 분노, 냉소, 반문, 책임 추궁을 쓴다.
- 욕설·비하·조롱 자체에 기대지 않는다.
- "우리 편이 무조건 선하다"보다 "이 기준으로 보면 누가 책임져야 하나"로 밀어붙인다.

### 기본 문장 구조

- "문제는 OOO가 아닙니다. 진짜 문제는 OOO입니다."
- "우파 반론은 이겁니다. 그런데 빠진 질문이 있습니다."
- "이게 실수라면 왜 하필 지금입니까?"
- "숫자는 앞서는데, 안심할 상황은 아닙니다."
- "표현의 자유냐, 혐오 장사냐. 어디까지 허용해야 합니까?"

## 3. 성과 목표

### 1차 목표: 팔로워 확보

- Hook 3초 유지율.
- 첫 10초 이탈률.
- 댓글 전환율.
- 공유율.
- 팔로우 전환율.

### 2차 목표: 수익화 기반

TikTok Creator Rewards 기준상 1분 이상 고품질 원본 콘텐츠, 재생 시간, 검색 가치, 참여가 중요하다. 따라서 하루 3개 중 최소 2개는 61초 이상으로 만들고, 모든 콘텐츠는 출처·해설·원본 이미지·TTS·편집을 포함한 자체 제작물이어야 한다.

### 3차 목표: 반복 가능한 자동화

매번 사람이 주제를 지정하지 않아도, 파이프라인이 최신 정치 쟁점에서 다음 요소를 스스로 산출해야 한다.

- 왜 이 주제인가.
- 좌파 지지층이 왜 반응하는가.
- 우파 반론은 무엇인가.
- 어떤 장면이 필요한가.
- 어떤 제목·해시태그·고정댓글이 댓글을 여는가.
- 업로드해도 되는가.

## 4. 주제 선정 엔진

### 수집 소스

- 최근 24~48시간 정치·사회 뉴스.
- 선거·지지율·정당·대통령·특검·재판·역사 이슈 검색어.
- 계정 기존 성과 피드백.
- TikTok Studio 지표가 존재하면 최우선 반영.
- 없으면 계정 스냅샷 기반 참고 피드백만 사용.

### 후보 클러스터

단일 키워드가 아니라 이슈 묶음으로 후보를 만든다.

- 인물 + 행위: 윤석열 + 위증 선고.
- 기업 + 역사: 스타벅스 + 5·18 탱크데이.
- 정당 + 프레임: 국민의힘 + 정부견제론.
- 여론조사 + 해석: 민주당 45%, 국민의힘 20~33%.
- 혐오 + 책임: 일베식 조롱 + 폐쇄·징벌 공론화.

### 점수 산식

각 후보는 100점 만점이 아니라 가중 합산으로 비교한다.

| 항목 | 기준 | 가중 |
| --- | --- | --- |
| 진보 반응성 | 분노·기억·책임·공정성 트리거가 있는가 | 25 |
| 현재성 | 24~48시간 내 보도·일정·논란인가 | 20 |
| 댓글 충돌성 | 좌우 반론이 명확히 갈리는가 | 18 |
| 출처 강도 | 2개 이상 신뢰 출처가 있는가 | 15 |
| 시각화 가능성 | 카드별 장면이 달라질 수 있는가 | 10 |
| 검색 가치 | 제목·해시태그 검색어가 살아있는가 | 8 |
| 정책 안전성 | 허위·명예훼손·선거절차 위험이 낮은가 | 4 |

### 탈락 조건

- 출처 1개뿐인데 단정형 분노 콘텐츠가 필요한 주제.
- 실존 인물 합성 없이는 설명이 어려운 주제.
- 숫자만 있고 갈등 구조가 없는 여론조사.
- 이미 같은 계정에서 비슷한 제목·장면으로 올린 주제.
- 선거 절차·결과를 미확인으로 단정해야 하는 주제.

## 5. 기획 워크플로우

### 01_signal_collection

최근 보도와 검색어를 모은다. 후보마다 `title`, `source`, `published_at`, `url`, `detected_terms`를 남긴다.

완료 기준:

- 후보 20개 이상.
- 강한 후보 5개 이상.
- 출처 없는 추정 후보는 별도 제외.

### 02_topic_clustering

중복 기사를 이슈 단위로 묶는다.

예시:

- "스타벅스 탱크데이", "서울재즈페스티벌 부스 취소", "오세훈 적당히 하라"는 하나의 이슈 축으로 묶는다.
- "리얼미터 60.5", "NBS 66", "갤럽 64"는 지지율 해석 축으로 묶는다.

### 03_topic_scoring

진보 반응성, 댓글 충돌성, 현재성, 출처 강도, 시각화 가능성으로 점수화한다.

완료 기준:

- 상위 7개 이슈.
- 하루 3개 슬롯별 1순위 후보.
- 예비 후보 2개.

### 04_opposition_frame

우파 반론을 먼저 적는다. 이것이 없으면 댓글이 약하다.

예시:

- "기업 실수에 대통령이 과하게 개입했다."
- "재판은 정치 보복이다."
- "지지율은 여론조사일 뿐이다."
- "표현의 자유를 막는 것이다."

그 다음 좌파 프레임을 붙인다.

- "역사 조롱이 단순 실수로 끝날 수 있나."
- "책임을 묻는 것과 정치 보복은 다르다."
- "숫자보다 결집 흐름이 위험 신호다."
- "표현의 자유와 혐오 장사는 다르다."

### 05_editorial_angle

각 콘텐츠는 한 문장 논지를 가져야 한다.

좋은 논지:

- "스타벅스 논란은 커피 문제가 아니라 역사 기억을 돈벌이에 태운 문제다."
- "지지율이 높을수록 민주당은 방심하면 안 된다."
- "윤석열 위증 선고의 핵심은 형량보다 계엄 정당화 서사의 붕괴다."

나쁜 논지:

- "요즘 정치가 시끄럽다."
- "댓글로 의견 남겨주세요."
- "팩트를 알아보자."

### 06_fact_pack

보도 사실, 당사자 주장, 반론, 미확인 영역을 분리한다.

형식:

- 확인된 사실.
- 보도된 주장.
- 반론.
- 아직 모르는 것.
- 콘텐츠에서 말하면 안 되는 것.

### 07_script_brief

스크립트는 카드뉴스 문구와 TTS 문구를 분리한다.

카드 문구:

- 짧고 강하게.
- 한 카드 1개 주장.
- 2줄 이내.

TTS:

- 감정은 있지만 과장하지 않는다.
- 숫자와 날짜를 한국어 발음으로 변환한다.
- 외래어는 한글 표기 우선.

### 08_visual_brief

이미지는 텍스트 뒤에 붙이는 배경이 아니라, 각 카드 주장에 맞춘 장면이어야 한다.

각 카드마다 다음을 만든다.

- visual_intent: 이 장면이 말하는 것.
- visual_anchor: 반드시 보여야 할 사물·공간.
- treatment: 실사풍 재구성, 풍자 테이블로, AiNo 진행자, 증거 책상, 댓글 대결.
- negative: 실존 인물 얼굴, 정당 로고, 실제 언론 화면, 읽히는 가짜 문서 금지.

### 09_script_variants

최소 3개 각본을 만든다.

- 분노형: 감정 진입이 강함.
- 근거형: 사실과 수치 중심.
- 충돌형: 우파 반론과 좌파 반박 중심.

상위 1개만 렌더링한다.

### 10_storyboard

기본 9~11카드.

| 카드 | 역할 | 권장 시간 |
| --- | --- | --- |
| 1 | 썸네일 Hook | 4~5초 |
| 2 | 왜 지금인가 | 6~7초 |
| 3 | 사건 요약 | 6~8초 |
| 4 | 첫 번째 근거 | 7~9초 |
| 5 | 우파 반론 | 6~8초 |
| 6 | 반박·빈칸 | 7~9초 |
| 7 | 책임 기준 | 7~9초 |
| 8 | 댓글 선택지 | 6~8초 |
| 9 | 정리 + 팔로우 CTA | 5~7초 |
| 10~11 | 필요한 경우 판세·출처 보강 | 6~8초 |

### 11_visual_generation

이미지 생성 순위:

1. Codex CLI 이미지 생성.
2. Gemini API fallback.
3. 로컬 fallback은 검수용만 허용, upload_ready 차단.

이미지 기준:

- 카드마다 장소·구도·상징이 달라야 한다.
- 실제 뉴스 사진처럼 보이되, 특정 실존 인물로 보이면 안 된다.
- 촬영감: 35mm/50mm, 강한 전경, 얕은 심도, 실제 조명, 낮은 채도.
- 풍자 장면은 캐릭터·상징물 중심으로 하고 정당 로고는 쓰지 않는다.

### 12_tts_generation

기본은 ElevenLabs Anna Kim 계열을 유지한다.

권장:

- model: eleven_multilingual_v2를 기준으로 유지하되, 한국어 전용·신규 모델은 ListModels로 실제 지원 여부 확인 후 비교.
- speed: 1.05~1.09.
- stability: 너무 낮으면 감정은 생기지만 흔들릴 수 있으므로 0.12 기준에서 샘플 비교.
- similarity: 기존 목소리 정체성 유지.
- style: 0.05~0.10.
- speaker boost: 활성.
- logging: 비활성 옵션이 사용 가능하면 비활성.

TTS 문장 규칙:

- "5·18"은 "오일팔" 또는 "오 일 팔 민주화운동"으로 별도 치환.
- "NBS"는 "엔비에스"로 치환.
- "60.5%"는 "육십 점 오 퍼센트"로 치환.
- "5월 28일"은 "오월 이십팔일"로 치환.
- 긴 문장은 쉼표와 마침표를 넣어 호흡을 만든다.

### 13_rendering

기본 스펙:

- 1080x1920.
- 30fps.
- H.264.
- 61~78초 기본.
- 카드 9~11장.
- 음성 싱크 기준 카드 길이 자동 보정.

움직임:

- 흔들림 금지.
- 지속적인 이미지 흔들림 금지.
- 기본은 정적 이미지 + 카드 전환.
- 허용: 1~1.5% 이하의 아주 약한 push-in, wipe, hard cut, flash, newspaper slide.

### 14_mobile_readability_gate

텍스트 기준:

- 썸네일 메인: 12~18자.
- 본문 카드: 한 줄 14~18자, 최대 2줄.
- 카드 전체 문구: 34~42자 권장.
- 출처·보조 문구: 18~28자.
- 가장 긴 단어가 박스 밖으로 나가면 실패.

폰트:

- Noto Sans CJK KR 또는 Pretendard 계열.
- 제목 78~96px.
- 본문 52~66px.
- 보조 36~44px.
- 자막 하단은 TikTok UI 영역을 피해 배치.

안전 영역:

- 상단 120px, 하단 320px, 우측 130px은 핵심 텍스트 회피.
- 댓글·좋아요 UI와 겹치면 실패.

### 15_policy_gate

반드시 차단:

- 선거 절차·결과·투표 방법에 대한 미확인 주장.
- 여론조사 수치 출처·조사일 미표기.
- 실존 정치인이 실제로 하지 않은 행동을 AI 이미지로 표현.
- 실제 언론 로고·정당 로고·공문서처럼 보이는 가짜 자료.
- 유료 광고, Promote, 브랜디드 정치 콘텐츠.

허용:

- 보도 근거가 있는 정치 논평.
- 출처와 반론을 포함한 이슈 해설.
- 공공 관심 사안에 대한 의견.
- AI 생성 이미지 고지.

### 16_metadata

제목:

- 45~80자.
- 검색어 1개 + 갈등어 1개 + 질문 1개.

캡션:

- 250~500자.
- 사실 1문장, 해석 1문장, 반론 1문장, 댓글 질문 1문장.
- "해당 이미지는 생성된 이미지입니다" 또는 TikTok AIGC 라벨 사용.

고정댓글:

- 선택지 2~3개.
- 팔로우 유도는 말미에 짧게.

해시태그:

- 8~12개.
- 단어 단위.
- 이슈 4개, 정체성 2개, 범용 2~4개.

## 6. 하루 3개 슬롯

정치 이슈는 너무 빨리 변하므로 7일치 전체 렌더링보다 7일치 기획 + 3일치 생성·예약이 더 안전하다.

| 시간 | 역할 | 기본 길이 | 목적 |
